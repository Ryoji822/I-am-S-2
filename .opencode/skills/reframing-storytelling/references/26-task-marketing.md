# Task Module: Marketing Content (マーケティング)

## 使いどころ

マーケティングコンテンツ、ランディングページ、提案文を作るときに使う。読後に「自分ごととして理解し、次の一歩を選べる」状態を作ることが目的。

## 構造

```
状況（共感ではなく事実）
→ 困りごと（具体）
→ 解決の筋（どう変わるか）
→ 根拠（数字/事例/仕組み）
→ 次の一歩（小さく）
```

## テンプレート

```text
あなたは{商材カテゴリ}の説明文を書きます。目的は「読者が自分ごととして理解し、次の一歩を選べる」ことです。

[読者]
- いまの状況: {状況}
- 困っていること: {困りごと}
- 期待する変化: {望む変化}

[構成]
- 状況（共感ではなく事実）→困りごと（具体）→解決の筋（どう変わる）→根拠（数字/事例/仕組み）→次の一歩（小さく）
- "根拠が弱い断言"は禁止。わからない点は「わからない」と書く。

[短い自己チェック]
- 何を売りたい文章かではなく、読者が何を決めたい文章になっているか
```

## 設計原則

- 読者の問題から始める。商品や技術からではない。
- 読者をヒーローに、提案者をガイドに位置づける。
- 誇張よりも「筋が通ること」と「早く理解できること」を優先する。
- 根拠が弱い断言は禁止。根拠がないなら「まだ分からない」と書く。

## 情報の非対称性の設計

- 読者が「自分にも関係ある」と気づく導入を置く
- すべてを冒頭で語らない。読者が自然に読み進められるよう段階的に開示する
- ただし、わざとらしい煽りや過度な期待値操作はしない

## 避けること

- 押しつけ
- 根拠のない断言
- 煽り
- 過剰な期待値操作
- 競合を貶す比較
- 「革新的」「画期的」「業界をリードする」等の空虚な形容

## quality gate

- 読者が「何を決めたい文章」になっているか（売りたい文章ではなく）
- 根拠が添えられているか
- 次の一歩が小さく具体的か
- 押しつけがましくないか
- 読者の状況認識が正確か

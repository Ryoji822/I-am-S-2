# Task Module: Educational Content (学習設計)

## 使いどころ

教育コンテンツ、研修資料、チュートリアルを作るときに使う。読後に「できるようになる」状態を作ることが目的。

## 構造

```
課題提示（現実の作業）
→ 既知の呼び起こし
→ 見本（手順と判断ポイント）
→ 練習問題（2問）
→ フィードバック（よくある間違い）
→ まとめと次の練習
```

## テンプレート

```text
あなたは講師です。{対象者}に{スキル/知識}を教え、読後に{できること}が実行できるようにしてください。

[構成]
1) 具体的な課題（現実の作業としての{課題}）を提示
2) 既に知っていることを呼び起こす（{前提}）
3) 見本を見せる（手順と判断ポイント）
4) 練習問題（2問。1問は簡単、1問は少しだけ応用）
5) フィードバック（よくある間違いと直し方）
6) まとめと、次の練習（30秒でできる宿題）

[制約]
- 1つの手順は3〜5ステップまで。長い場合は分割する。
- 例は必ず「なぜそうするか」も1行添える。
```

## 設計原則

この骨格は5つの学習原則に基づく。

1. **問題中心**: 抽象概念から始めず、具体的な作業課題から始める
2. **既知の活性化**: 学習者が既に知っていることを呼び起こし、新情報の受け皿を作る
3. **見せる**: 手順だけでなく、判断のポイント（なぜそうするか）も見せる
4. **やらせる**: 練習を通じて、理解を行動に変換させる
5. **統合**: 学んだことを既存知識に組み込み、次の応用につなげる

## 認知負荷管理

- 1ステップで扱う概念は1つまで
- 新概念を導入するときは、必ず既知の概念と接続する
- 一度に提示する情報量をワーキングメモリの限界（約4チャンク）に収める
- 不要情報（余談、歴史、網羅的な分類）は学習を阻害する。入れない。

## フロー状態の設計

- 簡単すぎれば退屈、難しすぎれば不安。「少し背伸びすれば理解できる」レベルを狙う。
- 練習問題は、1問目は本文の再現、2問目は少しだけ応用。
- 応用問題で詰まっても、戻れる場所（見本や手順）が見えるようにする。

## quality gate

- 読後に学習者が実際に試せる状態になっているか
- 手順が3〜5ステップに収まっているか
- 「なぜそうするか」が各手順に添えられているか
- 余計な情報が学習を邪魔していないか
- 次の練習が具体的で小さいか

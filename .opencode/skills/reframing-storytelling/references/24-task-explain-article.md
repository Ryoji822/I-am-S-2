# Task Module: Explanation Article (概念解説)

## 使いどころ

概念を説明する記事、解説文、理解を促すコンテンツを作るときに使う。読後に「概念を説明できる・判断できる」状態を作ることが目的。

## 構造

```
結論先出し（1〜2文の答え）
→ 用語の説明（最短）
→ なぜ重要か
→ 短い例（1つ）
→ まとめ（3行以内）
→ 次にやること（1行）
```

## テンプレート

```text
あなたは編集者です。{対象読者}に向けて、{テーマ}を「短く、必ず理解できる」ように説明してください。

[読者と目的]
- 読者: {対象読者。前提知識も明記}
- 読後にできること: {できるようになる行動を1つ}

[出力ルール]
- 最初に「答え（1〜2文）」を出す（結論先出し）。
- 次に「用語の説明（最短）」→「なぜ重要か」→「短い例（1つ）」→「まとめ（3行）」。
- 1段落1アイデア。段落の役割（理由/具体化/手順）を文頭で示す。
- 不要な情報は入れない（余談・歴史・網羅は不要）。
- 曖昧語（それ・これ・重要・最適など）を使う場合は、必ず指す対象を同じ文で言う。
- 最後に「読者が次にやること」を1行で示す。

[自己チェック]
- 読者の前提を超える語を定義せずに出していないか
- どの段落も中核メッセージに接続しているか
- "次に何をすればよいか"が残っているか
```

## 出力ルール

- 1段落1アイデアを守る
- 段落の役割（理由/具体化/手順）を読者が追えるようにする
- 不要な情報を入れない（余談・歴史・網羅は不要）
- 曖昧語を使う場合は、指す対象を同じ文で言う
- 専門語は先に平易な説明を置き、そのあとで用語名を1回だけ出す

## 認知負荷管理

- 1段落1メッセージ
- 段落間の関係（理由なのか、具体化なのか、手順なのか）が曖昧にならないようにする
- 段落間の論理関係を示す談話標識（「しかし」「つまり」「なぜなら」）を適切に使う

## quality gate

- 読後に概念を自分の言葉で説明できるか
- 定義されていない専門語がないか
- 因果の橋が切れていないか
- 「次にやること」が残っているか
- 余談が混じっていないか

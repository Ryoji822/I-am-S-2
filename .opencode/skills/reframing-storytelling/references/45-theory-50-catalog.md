# Theory: 50+50 Theory Catalog（理論カタログ）

## 目次
- 使い方
- Part A: ストーリーテリング50理論（storytelling_framework_rich由来）
- Part B: 要約・認知科学50理論（storytelling_summarization_framework由来）
- クロスリファレンス（重複・関連する理論の対応表）

## 使い方

品質問題の診断時に参照する。実行時にはロードしない。

1. 問題症状から関連理論を特定する
2. 理論の「AI示唆」欄から対策を読む
3. 対策を operational module（01, 30, 31等）の具体的指示に変換する

検索軸:
- **次元別**: 構造 / ギャップ / 共感 / 意味
- **クラスタ別**: A〜H（ストーリーテリング）、認知/言語/AI固有（要約）
- **問題別**: 因果断絶 / 共通知不成立 / 分布的収束 / 認知負荷超過 / 感情平坦化

## Part B: 要約・認知科学50理論

### テキスト理解・圧縮（10理論）

| # | 理論名 | AI示唆 |
|---|--------|--------|
| B1 | 抽出型 vs 抽象型要約 | AIは抽出型に偏る。抽象型（意味再構築）を強制 |
| B2 | マクロ構造理論（van Dijk） | まず全体のマクロ命題を定義→トップダウン |
| B3 | スキーマ理論 | 読者スキーマに合致する提示。あえてスキーマを裏切る展開も |

| B5 | 認知負荷理論（Sweller） | 短くするだけでなく構造化・視覚化 |
| B6 | 結束性と一貫性 | AIは結束性高いが一貫性欠如 |
| B7 | グライスの協調原理 | 「必要な量だけ語る」。冗長回避 |
| B8 | 関連性理論（Sperber & Wilson） | 読者にとっての関連性評価（Audience Design） |
| B9 | 推論の生成 | 余白を残すことが深い理解に繋がる |
| B10 | 因果的ネットワークモデル | 時系列→因果骨格の抽出・強調 |

## 概要

「理解しやすい」を「理解しやすいと判断できる条件を満たす」へ置き換えて運用する。根拠を定義して測ることで「絶対に」に近づける方針。

## クローズ法（穴埋めテスト）

文章の一部を空白にし、読者が文脈から適切な語を復元できるかで理解度を測定する。

**何を測るか**: 文脈統合のしやすさ。単語の短さだけでは拾えない要因（前提知識の不足、指示語の不明確さ、ぎこちない構文）を検出。

**実施方法**:
- 5〜7語ごとに穴を設定
- 短時間で実施可能
- 正答率50%以上 = 読みやすい（Taylor, 1953の基準）

**使いどころ**: 説明記事、教育コンテンツ、規程文

**AI要約への適用**: 元テキストの要約をクローズテスト化→読者に試す。復元率が低い箇所 = 共通知の不成立が発生している箇所。

## タスク成功率

読後に読者が想定タスクを実行できるかを直接測定。

**何を測るか**: 「読後にできること」の達成度

**計測項目**:
- 成功 / 失敗
- 完了時間
- 再試行回数
- エラー発生箇所

**使いどころ**: UXマイクロコピー、手順書、教育コンテンツ

**AI要約への適用**: 要約を読んだ後に「元テキストの主旨を3文で説明できるか」「判断が求められる場面で正しく判断できるか」をタスクとして設計。

## 主観評価（5段階）

読者の主観的評価。項目は3つに絞る（多いと雑になる）。

**推奨3項目**:
1. **明確さ（Clarity）**: 何を言っているか分かったか
2. **信頼（Trust）**: 信じてよいと感じたか
3. **次の手（Actionability）**: 次に何をすべきか分かったか

**使いどころ**: 全般。A/B比較で効く

**注意**: 主観評価は認知バイアスの影響を受ける。流暢さバイアス（滑らかな文章を高く評価する傾向）により、AIの「結束性は高いが一貫性が低い」文章が不当に高評価を得ることがある。

## 思考発話法（Think-Aloud）

読者にテキストを読みながら考えていることを声に出してもらう。

**何を測るか**: どこで詰まったかの理由

**実施方法**:
- 5〜8人でも主要な詰まり箇所は検出可能
- 定量評価とは別枠（質的データ）
- 録音→詰まり箇所をマッピング

**使いどころ**: 改善サイクル。プロトタイプ段階

**AI要約への適用**: 要約を読んでもらいながら「ここで何が言いたいか分からなくなった」箇所を特定 → 因果連鎖の断絶、前提の破綻、照応の不成立を直接検出。

## 読了時間・離脱率

定量的なエンゲージメント指標。

**何を測るか**: 「重い / だるい」を数値化

**計測方法**:
- 読了時間: 期待される読了時間からの乖離
- 離脱率: どのセクションで読者が離脱したか
- A/B比較が最も効果的

**使いどころ**: Web全般

**AI要約への適用**: 同一ソースの異なる要約を比較し、読了率・滞在時間を測定。感情曲線が平坦な要約（AP1: 均質な情報の洪水）は中間での離脱が多い。

## 4次元品質チェックリスト

4つの普遍的次元に基づく品質検証（storytelling-4d参照）:

| 次元 | チェック項目 | 判定基準 |
|------|-----------|---------|
| **構造** | 三段階変容が明確か | 開始時と終了時で認識が変化しているか |
| **構造** | セクション間に因果関係 | 「だからこそ」で接続されているか |
| **構造** | 読者が「今どこにいるか」把握 | ロードマップ・見出し・接続語が適切か |
| **ギャップ** | 冒頭にスキーマ裏切り | 最初の3文で「読み続けたい」か |
| **ギャップ** | 感情曲線にピーク | 最も印象的な瞬間を特定できるか |
| **ギャップ** | 情報の段階的開示 | 結論が冒頭ですべて述べられていないか |
| **ギャップ** | 読者の推論余地 | Show, Don't Tell が実践されているか |
| **共感** | 読者投影可能な主体 | 「あなた」「〇〇さん」等の具体的主体 |
| **共感** | 具体的人物と感情描写 | 五感を含む場面描写があるか |
| **共感** | 弱さ・試行錯誤 | 完璧な一方的提示になっていないか |
| **意味** | 逸話と考察の交互配置 | 抽象3文以上連続していないか |
| **意味** | 概念メタファー使用 | 日常経験への喩えがあるか |
| **意味** | スルーライン貫徹 | 全セクションが核心メッセージに収束 |
| **意味** | So What 明示 | 事実の後に読者にとっての意味 |

## テストワークフロー

```
指示を書く（読者・目的を固定）
       ↓
AIが下書き生成
       ↓
自動チェック（難易度・用語・長さ）
       ↓
人のチェック（5分テスト）
       ↓
合格? → Yes → 公開/配布
  ↓ No
指示を修正 → 再生成
```

「やさしい日本語」指針の「作る→書き換える→確認する」と同じ発想。

**5分テストの内容**:
1. クローズ法（2分）: 3箇所の穴埋め
2. 要旨再現（1分）: 「要するに何だったか」を口頭で説明
3. タスク（1分）: 「次に何をすべきか」を回答
4. 主観（1分）: 明確さ・信頼・次の手を5段階で回答

## 自動評価指標の限界

| 指標 | 何を測る | 限界 |
|------|---------|------|
| ROUGE | 元テキストとの表面的語彙一致 | 意味の圧縮やストーリー性を評価できない |
| BERTScore | 意味的類似度 | 共通知や認知負荷を考慮しない |
| 可読性スコア | 文の長さ・語の難しさ | 前提知識の不足や因果断絶を検出しない |

> 自動指標は「意味の圧縮」「ストーリー性」「読者の認知状態への適合」を完全には評価できない。人間の認知的評価（理解しやすさ）が最終的には重要。

jReadability（日本語文章難易度判定）は改稿のあたりをつけるには使えるが、過信は禁物。

## 実装への変換

1. evaluation-rubric（31）の quick gate 5問を最低限実施する
2. コンテンツ種別に応じた評価手法を選択する:
   - 説明記事 → クローズ法 + 主観評価
   - 教育 → タスク成功率 + 思考発話
   - UX → タスク成功率（成功/失敗 + 完了時間）
   - マーケティング → 主観評価 + 読了時間
3. A/B比較する場合: 小テスト正答率 / タスク成功率 / 主観クリアさを棒グラフ、5軸ならレーダーチャート
4. 自動指標は参考値として使い、最終判断は人間の認知的評価に委ねる
5. テストワークフローの「5分テスト」を品質ゲートとして組み込む
